# Red Flags

These thoughts usually mean you are rationalizing your way around the intended workflow:

| Thought | Reality |
|---------|---------|
| "This is just a simple question" | Questions are tasks. Check for skills. |
| "I need more context first" | Skill check comes before clarifying questions. |
| "Let me explore the codebase first" | Skills often define how to explore. |
| "I can check git/files quickly" | Quick checks easily become unstructured work. |
| "Let me gather information first" | The right skill often tells you what information matters. |
| "This doesn't need a formal skill" | If a skill exists, it probably captures a failure mode worth avoiding. |
| "I remember this skill" | Skills drift. Re-read the current version. |
| "This doesn't count as a task" | Action is task. |
| "The skill is overkill" | Small tasks are where assumptions sneak in. |
| "I'll just do this one thing first" | That is usually the point where discipline slips. |
| "This feels productive" | Productivity is not the same as correctness. |
| "I know what that means" | Knowing the concept is not the same as following the skill. |

When you hit one of these:

1. Stop
2. Re-check the minimal applicable set
3. Load the relevant skill
4. Continue only after the workflow is clear
